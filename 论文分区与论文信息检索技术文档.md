# 论文分区与论文信息检索技术文档

本文档用于指导 Agent 在其他项目中复用本项目的两项能力：

1. 从论文清单中抽取论文名称、期刊、年份，并按中科院分区分类。
2. 对指定分区论文补全题名、刊物信息、ISSN、卷期页码、影响因子、引用次数等信息。

适用场景：项目目录中有一份论文清单文件，例如 `.docx`，以及一份期刊分区或 JCR 信息表，例如 `.xlsx`。

## 一、输入文件约定

推荐项目根目录包含以下文件：

| 文件 | 用途 | 示例 |
| --- | --- | --- |
| 论文清单 `.docx` | 存放论文原始条目 | `庞善臣_2026-04-29.docx` |
| 分区表 `.xlsx` | 存放中科院分区与 JCR 数据 | `2025年度中国科学院及科睿唯安JCR期刊分区表.xlsx` |

本项目使用的 Excel 表结构如下：

| 工作表 | 字段 | 用途 |
| --- | --- | --- |
| `2025中国科学院分区表` | `期刊名称`、`2025分区`、`Top`、`Open Access` | 判断中科院分区和 TOP |
| `2025JCR分区` | `期刊名`、`ISSN`、`eISSN`、`Category`、`2024JIF` | 补全 ISSN、影响因子、SCIE/SSCI 类型 |

如果其他项目中的工作表名称或字段名不同，Agent 需要先读取表头，再建立字段映射。

## 二、依赖与运行环境

推荐使用 Python 3，优先使用标准库解析 `.docx`，使用 `openpyxl` 读取 `.xlsx`。

依赖：

```bash
pip install openpyxl
```

网络信息检索可使用公开 API：

| 来源 | 用途 |
| --- | --- |
| Semantic Scholar Graph API | DOI 批量检索题名、期刊、卷页、总被引 |
| OpenAlex API | DOI 或题名检索论文元数据、公开总被引 |
| Crossref API | DOI 检索出版元数据、卷期页码、出版商 |

注意：公开 API 通常不能提供严格意义上的“他引次数”。如果用户要求“他引次数”，Agent 应明确标注公开 API 获取的是“总被引”，不能等同于 Web of Science、知网、万方等数据库的他引口径。

## 三、分区检索技术

### 目标

从 `.docx` 论文清单中提取每条论文的：

- 原始条目
- 论文名称
- 期刊名称
- 发表年份
- 中科院分区
- 是否 TOP

并输出为 Markdown，一级标题按以下顺序排列：

```text
一区（TOP）
一区
二区（TOP）
二区
三区
四区
未检索到中科院分区
```

### 解析 `.docx`

`.docx` 本质是 zip 文件，可直接读取 `word/document.xml`。

核心思路：

1. 解压读取 `word/document.xml`。
2. 遍历所有 `w:p` 段落。
3. 合并段落中的所有 `w:t` 文本。
4. 使用正则识别编号论文条目。

示例识别规则：

```python
re.match(r"^(\d+)\.\s*(.+)", paragraph)
```

只处理形如 `1. 作者, 题名[J], 期刊, 年份...` 的段落。

### 期刊名称抽取

常见格式：

```text
作者, 论文题名[J], 期刊名称, 年份, 卷(期), 页码, DOI
```

推荐抽取策略：

1. 优先识别 `[J], 期刊名称, 20xx`。
2. 兼容异常格式，例如：
   - `题名,期刊名称[J], Volume 161, 2025...`
   - `题名,Knowledge-Based Systems[J], Volume 323...`
   - `作者[J], 题名[J], 2024...`
3. 如果期刊名缺失或位置异常，可使用 DOI 前缀反推常见 Elsevier 期刊：
   - `j.engappai` -> `ENGINEERING APPLICATIONS OF ARTIFICIAL INTELLIGENCE`
   - `j.patcog` -> `PATTERN RECOGNITION`
   - `j.knosys` -> `KNOWLEDGE-BASED SYSTEMS`
   - `j.eswa` -> `EXPERT SYSTEMS WITH APPLICATIONS`
   - `j.measurement` -> `MEASUREMENT`

### 期刊名称规范化

不同来源可能存在大小写、连字符、`&`、斜杠、缩写差异。匹配前必须规范化。

推荐规范化规则：

```python
def norm(s):
    s = str(s or "").replace("&", " and ")
    for ch in ["‐", "‑", "–", "—", "－", "/"]:
        s = s.replace(ch, " ")
    s = unicodedata.normalize("NFKD", s)
    s = re.sub(r"[^a-zA-Z0-9]+", " ", s).lower()
    return re.sub(r"\s+", " ", s).strip()
```

### 别名匹配

必须支持期刊别名，否则容易漏匹配。

示例：

```python
alias_pairs = {
    "Brief in bioinformatics": "BRIEFINGS IN BIOINFORMATICS",
    "Cluster Comput": "Cluster Computing-The Journal of Networks Software Tools and Applications",
    "Future Generation Computer Systems": "Future Generation Computer Systems-The International Journal of eScience",
    "Applied Sciences": "Applied Sciences-Basel",
    "IEEE/CAA Journal of Automatica Sinica": "IEEE-CAA Journal of Automatica Sinica",
}
```

匹配顺序建议：

1. 精确规范化匹配。
2. 别名匹配。
3. DOI 期刊代码匹配。
4. 高阈值模糊匹配，例如 `difflib.get_close_matches(..., cutoff=0.94)`。
5. 仍无法匹配则放入 `未检索到中科院分区`。

### 分组规则

根据 `2025分区` 与 `Top` 字段生成标题：

| 分区 | Top | 输出标题 |
| --- | --- | --- |
| 1 | 是 | `一区（TOP）` |
| 1 | 否 | `一区` |
| 2 | 是 | `二区（TOP）` |
| 2 | 否 | `二区` |
| 3 | 任意 | `三区` |
| 4 | 任意 | `四区` |

注意：Excel 中的中文可能因编码显示为 `ÊÇ`、`·ñ`。Agent 应把 `是` 和 `ÊÇ` 都视为 True。

```python
def is_yes(value):
    return str(value).strip() in {"是", "ÊÇ", "Yes", "YES", "Y", "1", "True", "true"}
```

## 四、论文信息检索技术

### 目标字段

对指定分区，例如 `一区（TOP）`，补全以下字段：

| 字段 | 来源优先级 |
| --- | --- |
| 著作或论文名称 | DOI API > 原始条目解析 |
| 出版单位或刊物名称 | JCR 表 > DOI API > 原始条目 |
| 国际刊号 | JCR 表中的 `ISSN`、`eISSN` |
| 期号、起止页码 | 原始条目 > DOI API |
| 所有著、作者姓名 | 原始条目，保留 `*` |
| 出版或发表年度 | 原始条目 > DOI API |
| 期刊影响因子 | JCR 表中的 `2024JIF` |
| 他引次数 | 优先用户提供的权威数据库；否则公开 API 总被引并明确说明 |

### DOI 抽取

推荐规则：

```python
re.search(r"10\.\d{4,9}/[^\s,，]+", raw, re.I)
```

抽取后清理末尾标点：

```python
doi = re.sub(r"[\s\]\)）.,，。；;]+$", "", doi)
```

### 题名抽取

优先使用 DOI API 返回的 `title`。若无 DOI，则从原始条目中抽取。

原始条目抽取策略：

1. 取第一个 `[J]` 前的内容。
2. 用逗号分割。
3. 从第一个包含冒号或较长英文短语的片段开始，视为题名。
4. 对异常格式 `作者[J], 题名[J]`，取两个 `[J]` 之间的题名。

### 作者抽取

作者信息以原始条目为准，因为原始条目中可能已用 `*` 标注通讯作者。

推荐策略：

1. 定位题名开始位置。
2. 题名前面的逗号分隔内容视为作者列表。
3. 不使用 API 作者覆盖原始作者，以免丢失 `*`。

### 期刊 ISSN、影响因子、SCIE/SSCI 类型

从 `2025JCR分区` 工作表按期刊名匹配。

字段映射：

| 输出字段 | Excel 字段 |
| --- | --- |
| 刊物名称 | `期刊名` |
| ISSN | `ISSN` |
| eISSN | `eISSN` |
| 期刊影响因子 | `2024JIF` |
| SCI/SSCI 类型 | `Category` 中包含 `SCIE` 或 `SSCI` |

类型判断：

```python
def source_type(category):
    if "SSCI" in category:
        return "SSCI"
    if "SCIE" in category:
        return "SCIE"
    if "ESCI" in category:
        return "ESCI"
    return "未知"
```

### 引用次数处理

如果用户未提供 Web of Science、知网等权威数据库导出结果，Agent 不应声称获得了严格“他引次数”。

推荐表述：

```text
公开源未提供严格他引；Semantic Scholar总被引 N（SCIE）
```

或：

```text
公开源未提供严格他引；OpenAlex总被引 N（SCIE）
```

如果公开 API 也无法获取：

```text
公开源未取到他引（SCIE）
```

```

```

## 六、迁移到其他项目的步骤

Agent 在新项目中应按以下顺序执行：

1. 列出项目根目录文件，确认 `.docx` 和 `.xlsx` 文件。
2. 读取 `.xlsx` 的工作表名称和前几行，确认字段结构。
3. 读取 `.docx` 的段落文本，确认论文条目格式。
4. 建立期刊分区表索引：
   - key 为规范化期刊名
   - value 包含分区、Top、Open Access
5. 建立 JCR 信息表索引：
   - key 为规范化期刊名
   - value 包含 ISSN、eISSN、Category、JIF
6. 从论文清单中抽取论文条目。
7. 对每条论文抽取期刊、年份、DOI。
8. 使用精确匹配、别名匹配、DOI 期刊代码、模糊匹配进行分区检索。
9. 输出 `output.md`。
10. 如果用户要求详细信息：
    - 从目标分区抽取条目。
    - 使用 DOI 批量查询 Semantic Scholar。
    - 对缺 DOI 或未命中的条目，用 OpenAlex/Crossref 按题名补查。
    - 用 JCR 表补 ISSN、JIF、SCIE/SSCI 类型。
    - 输出详细 Markdown 表格。

## 七、Agent 执行注意事项

1. 保留原始条目，不要擅自改写作者、题名或通讯作者标注。
2. 原始条目中 `*` 通常表示通讯作者，详细表中必须保留。
3. 分区匹配要可解释；无法匹配的条目不要丢弃，应放入 `未检索到中科院分区`。
4. 会议论文、中文期刊、预印本通常无法在中科院期刊分区表中匹配，属于正常情况。
5. 公开 API 的引用数是总被引，不是严格他引。必须在输出中说明。
6. 2025 或 2026 的在线发表论文可能没有完整卷期页码，应填 `未知` 或文章号。
7. 如果 Excel 表字段变化，先读取表头，不要硬编码工作表结构。
8. 如果期刊名称匹配到多个候选，优先选择完整精确名称，避免将 `Pattern Recognition` 错配为 `Pattern Recognition Letters`。
9. 对短期刊名，例如 `Measurement`、`Cells`、`Information`，不要用全文包含匹配直接判定，应优先使用 `[J], 期刊, 年份` 结构抽取。
10. 写入结果前，统计总条目数、已匹配数、未匹配数，并抽查输出标题和表格行数。

docx_entries = read_docx_entries(docx_path)

## 九、建议封装为项目工具

如果需要在多个项目中长期使用，建议将逻辑封装为：

```text
scripts/
  paper_partition.py
  paper_metadata.py
  paper_common.py
```

推荐命令：

```bash
python scripts/paper_partition.py --docx 论文清单.docx --xlsx 分区表.xlsx --out output.md
python scripts/paper_metadata.py --input output.md --section "一区（TOP）" --xlsx 分区表.xlsx --out 一区TOP详细信息.md
```

这样其他项目只需要复制 `scripts/` 和本文档，再替换输入文件即可复用。
